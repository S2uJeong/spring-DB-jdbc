## 체크와 언체크예외
- 의미상 차이 : 예외를 처리 할 수 없을 때 던지는 것을 생략가능한지 생략불가한지
- 컴파일러 입장에서 체크하는지가 체크/언체크 예외임

### 체크예외
- `Exception`
- `try-catch` or `throws` 를 하지 않으면 컴파일러 에러가 발생한다.
- 장점 : 훌륭한 안전장치
- 단점 : 번거롭다. 많은 라이브러리를 쓸땐 더더욱,, + 의존관계에 따른 단점 (어차피 내가 처리 못해!!!)

### 언체크예외
- `RuntimeException`
- 예외를 잡아서 처리하지 않아도 throws 생략가능
